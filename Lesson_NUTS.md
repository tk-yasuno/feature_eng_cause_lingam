- ESS (tail): 493 → 1627（**+230%**）

**結論**: **本番設定（2000/1000）は論文投稿に十分な品質**

### 4.2 軽量設定の有効性

✅ **メリット**:
- **実行時間短縮**: 約20分で完了（本番設定の約1/2）
- **早期検証**: パイプライン全体の動作確認が可能
- **Divergence無し**: モデル構造とパラメータ化は適切

⚠ **デメリット**:
- **ESS不足**: 一部パラメータでESS (bulk) < 400
- **R-hat微妙**: log_lambda[3-5]で1.03-1.04（理想は<1.01）
- **信頼区間の精度**: 推論精度がやや低い可能性

### 4.3 中間状態の推定困難性（解決済み）

**現象（軽量設定）**:
- log_lambda[3-5]（状態4-6）でR-hatが1.03-1.04と高い
- これらの状態でESS (bulk)が最小（371-448）

**原因**:
- 中間状態（3-6）は健全と劣化の境界領域
- トランジション頻度が低く、データが少ない
- パラメータ識別性が低下

**対策（本番設定）と効果**:
- **n_draws**: 1000 → 2000（サンプル数2倍）
- **n_tune**: 500 → 1000（適応期間2倍）
- **結果**: R-hat 1.04 → 1.01、ESS 371 → 744（✅ 成功）

### 4.4 非中心化パラメータ化の成功

```python
# 非中心化（実装済み）
u_raw = pm.Normal('u_raw', mu=0, sigma=1, shape=n_pumps)
sigma_u = pm.HalfNormal('sigma_u', sigma=5)
u = pm.Deterministic('u', u_raw * sigma_u)
```

✅ **効果**:
- Divergence 0件（軽量・本番ともに完璧）
- sigma_uの収束良好（本番: R-hat 1.00、ESS 1414）
- u_rawとsigma_uの相関を回避

### 4.5 実行時間の実測値

| 設定 | n_draws | n_tune | 実行時間 | 速度 (draws/秒) |

## 6. トラブルシューティング

### 問題1: Divergences > 1%

**原因**: パラメータ化の問題、step size不適切

**対策**:
1. **target_accept上げる**: 0.95 → 0.99
2. **非中心化確認**: u = u_raw * sigma_u（実装済み）
3. **n_tune増やす**: 1000 → 2000

### 問題2: R-hat > 1.05

**原因**: チェーン間の収束不足

**対策**:
1. **n_draws増やす**: 2000 → 3000
2. **n_chains増やす**: 8 → 16（要16コアCPU）
3. **初期値確認**: 極端な値から開始していないか

### 問題3: ESS < 100

**原因**: 自己相関が高い、混合不良

**対策**:
1. **n_draws大幅増**: 2000 → 5000
2. **モデル簡略化**: 共変量減らす、状態数減らす
3. **prior見直し**: 情報的priorに変更

### 問題4: メモリ不足

**原因**: 8チェーン × n_draws × 124パラメータ

**対策**:
1. **n_chains減らす**: 8 → 4
2. **保存間引き**: idata_kwargsで指定
3. **分割実行**: グループ別に推定

**解決策**:

**PyMC 5.x + ArviZ 1.x**:
```python
# ❌ 旧API（削除された）

# ✅ 新API
trace.to_netcdf("trace.nc")
```

**収束診断の型変換**:
